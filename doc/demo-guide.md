# WorkBrain 演示与录屏指南

这份指南用于生成可重复的演示数据，并录制一段能够向面试官说明产品价值、工程能力和安全边界的视频。

## 录制前检查

1. 启动完整环境：

   ```bash
   docker compose up --build -d
   ```

2. 检查所有长期运行服务均为 healthy：

   ```bash
   docker ps --filter 'name=workbrain-'
   ```

3. 检查 API：

   ```bash
   curl --fail http://localhost/api/health/live
   curl --fail http://localhost/api/health/ready
   ```

4. 执行生产冒烟测试：

   ```bash
   .venv/bin/python scripts/production_smoke.py --base-url http://localhost
   ```

5. 生成一套独立的作品集演示数据：

   ```bash
   .venv/bin/python scripts/seed_demo_data.py --base-url http://localhost
   ```

脚本输出三种角色的用户名、随机共享密码、组织编号、知识库编号、服务目录和三种状态的申请。输出包含登录凭据，不要提交到 Git、Issue 或公开录屏字幕。

演示数据包括：

- 一个 admin、一个 approver、一个 member
- 一个企业知识库
- VPN、软件安装、电脑更换三个服务项目
- 一条已批准申请
- 一条有拒绝理由的已拒绝申请
- 一条待审批申请
- 创建、批准和拒绝审计记录

## 演示资料

需要展示文档处理和 RAG 时，上传：

```text
evals/enterprise_it_service_handbook.md
```

该文件是专门编写的非敏感演示资料，可以安全用于录屏。上传会触发 OpenAI Embeddings；RAG 回答会调用 DeepSeek，因此正式录制前应确认模型密钥、额度和网络均可用。

建议问题：

```text
申请 VPN 远程访问服务需要填写什么？
```

预期效果：

- 回答只使用企业资料。
- 回答包含 `[S1]` 等引用。
- 页面展示来源文档、分块预览和相关度。

资料不足拒答问题：

```text
公司食堂今天供应什么午餐？
```

预期效果是返回资料不足提示，而不是编造答案。

## 5 分钟演示脚本

### 0:00～0:30：产品定位

画面：登录页和工作台。

讲解：

> WorkBrain 是一个企业知识库与 IT 服务申请助手。它不只是一个聊天界面，而是把组织权限、异步文档处理、带引用 RAG、服务申请、审批审计和 Agent 安全确认整合成完整业务闭环。

### 0:30～1:10：多组织与角色权限

画面：组织与成员页面。

操作：

1. 展示当前组织选择器。
2. 展示 admin、approver、member 三种角色。
3. 切换到另一个组织，说明资源随组织上下文变化。

讲解：

> JWT 只证明用户身份。每个企业接口还要求 `X-Organization-ID`，后端会重新校验成员关系，并在数据库查询中限制 `organization_id`。前端隐藏按钮不能替代后端授权。

### 1:10～2:10：文档处理与带引用 RAG

画面：文档管理和 AI 问答。

操作：

1. 上传 `evals/enterprise_it_service_handbook.md`。
2. 展示 queued、running、succeeded 处理状态。
3. 使用 approver/admin 发布文档。
4. 提问 VPN 申请问题。
5. 展开 `[S1]` 引用。
6. 提问食堂问题，展示资料不足拒答。

讲解：

> 上传请求只保存文件和任务记录，Celery Worker 异步完成解析、切块和向量化。RAG 只检索当前组织、当前知识库内已发布的文档，并结合语义分数、词法匹配和阈值过滤。没有足够证据时不会调用模型编造答案。

### 2:10～3:15：IT 服务目录与审批

画面：IT 服务目录、我的申请和审批工作台。

操作：

1. 以 member 登录。
2. 创建一条服务申请。
3. 展示初始状态是 pending。
4. 以申请人打开审批页，展示不能审批自己的申请。
5. 以 approver 登录并批准申请。
6. 返回申请详情，展示 create 和 approve 审计事件。

讲解：

> 普通成员只能查看自己的申请，approver 和 admin 可以查看当前组织的申请，但申请人不能审批自己，结束状态也不能重复流转。每次状态变化都有操作者、前后状态、原因和时间。

### 3:15～4:10：Agent 确认

画面：IT 服务助手。

输入：

```text
我要申请 VPN，标题是远程办公 VPN，用于下周在家处理线上故障。
```

操作：

1. 展示 Agent 识别出的服务、标题和说明。
2. 强调确认卡片出现时数据库还没有新增申请。
3. 点击确认。
4. 在“我的申请”中查看新建的 pending 申请。

讲解：

> Agent 不能直接执行高风险写操作。第一阶段只生成确认内容和一次性令牌；第二阶段由用户主动提交令牌，后端重新校验用户、组织和服务状态。令牌只保存哈希并保证幂等，这就是两阶段确认。

### 4:10～4:40：跨组织隔离与可观测性

画面：切换组织、任务状态、申请审计或 Agent Trace。

讲解：

> 跨组织隔离不是依赖 Prompt，而是在身份依赖、策略函数和每条资源查询中落实。系统同时记录 Request ID、后台任务、RAG 查询、Tool Call、Agent Trace 和业务审计，便于定位一次请求经过了哪些组件。

### 4:40～5:00：工程收尾

画面：README 的架构图和测试命令。

讲解：

> 项目使用 PostgreSQL、pgvector、Redis、Celery、React、FastAPI、Caddy 和 Docker Compose。后端有 300 级别的自动化测试，前端覆盖核心交互，同时具备迁移检查、健康检查和生产冒烟测试。

## 90 秒精简演示

时间不足时只展示：

1. 组织选择器和三种角色。
2. RAG 回答及 `[S1]` 引用。
3. Agent 确认卡片。
4. approver 批准申请。
5. 申请详情中的审计记录。

一句话收尾：

> WorkBrain 展示的重点，是如何把 LLM 放进有权限、有审计、有确认、有测试的企业业务系统，而不是只调用一次模型 API。

## 录屏建议

- 使用 1920×1080 或 1440×900，浏览器缩放保持 100%。
- 关闭通知、个人标签页和密码管理器弹窗。
- 使用全新的演示用户名，不展示真实个人数据。
- 先完整演练一次，再录制最终版本。
- 操作之间停留 1～2 秒，让观众看清状态变化。
- 视频中不要展示 `.env`、JWT、确认令牌、数据库 URL 或模型密钥。
- 如果模型网络不稳定，提前录制一次成功的 RAG 和 Agent 流程作为备用。

## 演示边界

- 演示数据脚本会写入目标环境，应该在本地、预发布或允许保存演示数据的作品集环境运行。
- 当前没有组织和用户的级联删除接口，不要在正式客户数据环境反复执行脚本。
- Docker Compose 展示的是单机部署能力，不应描述成 Kubernetes 或多区域高可用。
- MCP Server 是只读学习 Demo，不属于生产 Agent 链路。
- 跨组织隔离应展示前端切换和后端测试，不能只用“Prompt 不允许越权”解释。
